# HappyCow Scraper: Multi-Page Type Crawling Fix PRD

## Problem Statement

The HappyCow scraper successfully crawls city listing pages but fails completely when attempting to crawl individual restaurant pages. Analysis reveals that the scraper uses a one-size-fits-all approach with city listing CSS selectors for ALL page types, causing crawl4ai to timeout waiting for elements that don't exist on restaurant pages.

**Current Issues:**
- ✅ City listing pages work (e.g., `/north_america/usa/texas/dallas/`)
- ❌ Individual restaurant pages fail (e.g., `/reviews/restaurant-name-123/`)
- ❌ Using wrong CSS selectors: `.card-listing, .venue-list-item` on restaurant pages
- ❌ crawl4ai returns `Success: False` for all restaurant pages
- ❌ No page type detection or conditional crawling logic

## Success Criteria

### Primary Success Criteria
1. **City Listing Pages**: Continue to work as they currently do
   - Successfully fetch HTML content (800K+ characters)
   - Find restaurant listings using appropriate selectors
   - Extract restaurant URLs for individual scraping

2. **Individual Restaurant Pages**: Successfully crawl and extract data
   - crawl4ai returns `Success: True` for restaurant pages
   - Extract restaurant details (name, address, phone, reviews, etc.)
   - Handle restaurant-specific page structure and selectors

3. **Page Type Detection**: Automatically detect page type and use appropriate settings
   - Detect city listing pages vs individual restaurant pages
   - Apply correct CSS selectors based on page type
   - Use appropriate timeout and crawling strategies per page type

### Technical Success Criteria
- **Zero Breaking Changes**: Existing city listing functionality remains intact
- **Performance**: Restaurant page crawling completes within 30 seconds per page
- **Reliability**: 95%+ success rate for both page types under normal conditions
- **Error Handling**: Clear error messages distinguishing page type vs crawling failures

## Technical Requirements

### Page Type Detection
- **URL Pattern Recognition**: Detect page type based on URL patterns
  - City pages: `/north_america/usa/state/city/` or similar geographic patterns
  - Restaurant pages: `/reviews/restaurant-name-id/` or `/reviews/restaurant-name-id/update`
  - Handle edge cases and variations in URL structure

### CSS Selector Strategy
- **City Listing Pages**: 
  - Primary selectors: `.card-listing, .venue-list-item`
  - Fallback selectors: `.venue-item, .restaurant-item`
  - Error detection: `.no-results, .captcha`

- **Restaurant Pages**:
  - Primary selectors: `.restaurant-name, .venue-details, h1`
  - Content selectors: `.review-content, .venue-info, .contact-info`
  - Error detection: `.not-found, .error-message`

### Crawl4AI Configuration
- **Conditional Configuration**: Different crawl4ai settings per page type
- **Timeout Management**: Appropriate timeouts for each page type
- **Wait Strategies**: Page-specific wait conditions and selectors
- **Error Recovery**: Retry logic with different selectors if initial attempt fails
